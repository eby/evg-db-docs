= config.org_unit_setting_type_log
:description: Table reference for config.org_unit_setting_type_log

xref:schemas/config.adoc[↑ config schema]


Org Unit setting Logs

This table contains the most recent changes to each setting 
in actor.org_unit_setting, allowing for mistakes to be undone.
This is NOT meant to be an auditor, but rather an undo/redo.


[CAUTION]
====
*Data-Modifying Triggers:* This table has BEFORE ROW trigger(s) that modify row data before write. Values you INSERT or UPDATE may differ from what is actually stored. See the <<triggers,Triggers>> section below.
====

[WARNING]
====
*Cascading Deletes:* Deleting rows from this table will cascade to: xref:tables/actor/actor_org_unit.adoc[actor.org_unit].
====

[WARNING]
====
*Deferrable Constraints:* The following FK constraints are deferrable — they are checked at transaction end, not statement end: `config_org_unit_setting_type_log_fkey`, `org_unit_setting_type_log_field_name_fkey`.
====

=== Trigger Bodies

==== limit_logs_oust

*Function:* xref:functions/evergreen.adoc#limit_oustl[`evergreen.limit_oustl()`] +
*Timing:* BEFORE INSERT OR UPDATE ROW

[CAUTION]
====
This trigger modifies the row before it is written (returns a modified `NEW`).
====

[source,plpgsql]
----

    BEGIN
        -- Only keeps the most recent five settings changes.
        DELETE FROM config.org_unit_setting_type_log WHERE field_name = NEW.field_name AND org = NEW.org AND date_applied NOT IN 
        (SELECT date_applied FROM config.org_unit_setting_type_log WHERE field_name = NEW.field_name AND org = NEW.org ORDER BY date_applied DESC LIMIT 4);
        
        IF (TG_OP = 'UPDATE') THEN
            RETURN NEW;
        ELSIF (TG_OP = 'INSERT') THEN
            RETURN NEW;
        END IF;
        RETURN NULL;
    END;
----
